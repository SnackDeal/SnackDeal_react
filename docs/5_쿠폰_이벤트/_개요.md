# 쿠폰/이벤트

> 확정 명세 문서가 없어 `src/types/coupon.ts`, `src/stores/couponStore.ts`,
> `src/pages/user/EventList.tsx`, `src/pages/user/EventDetail.tsx`,
> `src/pages/user/CouponBox.tsx`를 기준으로 정리한 작업용 개요입니다.

## 관련 테이블 / 타입 (`src/types/coupon.ts`)

- `coupon_board` → `CouponBoard` — **이벤트 게시판**. `start_at`~`end_at`은 "게시일"(화면에
  이벤트가 노출되는 기간)을 의미한다. 사용자 화면의 "이벤트"는 이 테이블을 가리킨다.
- `coupon` → `Coupon`
  - `discount_type`: `'FIXED' | 'PERCENT'`
  - `issue_type`: `'EVENT' | 'SIGNIN'` — `EVENT`는 이벤트(쿠폰 게시판)와 연결되어
    `coupon_board_id`가 **필수**, `SIGNIN`(가입 축하 등)은 `coupon_board_id`가 `null` 허용.
  - `valid_from`~`valid_until`은 "쿠폰 받기(다운로드)가 열려 있는 기간" — `coupon_board`의
    게시 기간과 다른 개념이니 혼동하지 말 것.
  - `total_quantity` / `issued_quantity`로 선착순 수량 관리.
- `user_coupon` → `UserCoupon` — 회원이 실제로 받은(보유한) 쿠폰. `status`:
  `'ACTIVE' | 'USED' | 'EXPIRED'`. 쿠폰함(`/mypage/coupons`) 목록이 이 테이블 기준.

## 화면 / 라우트

| 라우트 | 페이지 | 비고 |
|---|---|---|
| `/event` | `EventList` | `coupon_board` 목록 (진행중/종료 등) |
| `/event/:id` | `EventDetail` | 이벤트 상세, 쿠폰 다운로드 버튼 |
| `/mypage/coupons` | `CouponBoxPage` → `CouponBox` | 보유 쿠폰(`user_coupon`) 목록, 상태별 표시 |

체크아웃(`/checkout`)에서 `user_coupon`을 선택해 주문 할인에 적용한다 —
[`../4_주문_결제_배송/_개요.md`](../4_주문_결제_배송/_개요.md)의 `Order.user_coupon_id` 참고.
