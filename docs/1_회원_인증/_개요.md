# 회원/인증

> 이 문서는 별도의 확정 명세가 아직 없어 현재 코드(`src/types/member.ts`, `src/stores/authStore.ts`,
> `src/lib/mockAuth.ts`, `src/pages/user/Login.tsx` 등)를 기준으로 정리한 작업용 개요입니다.
> 실제 명세가 확정되면 이 문서를 그 내용으로 대체하세요.

## 관련 테이블 / 타입

- `member` → `Member` (`src/types/member.ts`)
  - `gender`: `'MALE' | 'FEMALE'`
  - `status`: `'ACTIVE' | 'INACTIVE' | 'DELETED'`
  - `role`: `'USER' | 'ADMIN'`
  - `id`는 BIGINT(number), `birth`/`last_login`/`created_at`은 ISO 날짜 문자열
- `email_verification` → `EmailVerification` — 이메일 인증 코드/토큰 (회원가입 시 이메일 인증 플로우용)

## 인증 방식 (mock)

- 실제 백엔드 없이 `src/lib/mockAuth.ts`가 로그인/회원가입을 흉내낸다. 토큰 검증 로직은 없고,
  성공 시 `AuthResult`(`access_token`, `refresh_token`, `user`)를 그대로 세션에 저장한다.
- 세션 저장은 `src/stores/authStore.ts` (zustand + `persist`, localStorage 키: `snackdeal-auth`).
  `isAdmin`은 `user.role === 'ADMIN'`으로 파생.
- 관리자 로그인은 별도 스토어 `src/stores/adminAuthStore.ts`를 사용한다 (`AdminSession`,
  role은 `'ADMIN' | 'SUPER_ADMIN'`) — 회원 인증과 완전히 분리된 세션.
- 실제 이메일 발송은 콘솔 로그로 mock 처리한다 (`CLAUDE.md` 참고).

## 화면 / 라우트

| 라우트 | 페이지 | 비고 |
|---|---|---|
| `/login` | `LoginPage` → `Login` | 레이아웃 없음 |
| `/signup` | `SignupPage` → `Signup` | 레이아웃 없음 |
| `/admin/login` | `AdminLoginPage` → `AdminLogin` | 관리자 전용, 회원 로그인과 별개 |
| `/mypage` | `MyPage` | 로그인 필요 |

인증이 필요한 사용자 라우트(`/cart`, `/mypage/*` 등)는 `authStore`에 세션이 없으면 로그인
페이지로 리다이렉트된다 (구현 위치: 각 페이지 또는 라우트 가드 — 코드에서 실제 가드 위치를
확인할 것).
