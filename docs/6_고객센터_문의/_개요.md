# 고객센터/문의/챗봇

> 확정 명세 문서가 없어 `src/types/cs.ts`, `src/stores/csStore.ts`,
> `src/pages/user/Notice.tsx`, `src/pages/user/QnaList.tsx`, `src/pages/user/QnaNew.tsx`를
> 기준으로 정리한 작업용 개요입니다. **챗봇 관련 코드/화면은 현재 저장소에서 발견되지 않음**
> (미구현 또는 별도 예정) — 착수 전 팀 확인 필요.

## 관련 테이블 / 타입 (`src/types/cs.ts`)

- `notice` → `Notice` — 공지사항. `is_pinned`(상단 고정), `deleted_at`(소프트 삭제).
- `faq` → `Faq` — 자주 묻는 질문. `type`: `'ORDER' | 'SHIPPING' | 'PRODUCT' | 'OTHER'`
  (문의 유형과 동일한 enum 공유).
- `qna` → `Qna` — 1:1 문의. `type`은 `Faq`와 동일한 4종. `is_answered`는 **불리언**이며
  별도 상태 enum이 아니다(답변 여부만 표시).
- `qna_answer` → `QnaAnswer` — `qna_id`가 UNIQUE, 즉 **문의 1건당 답변 1건**만 존재
  (스레드형 다중 답변 아님).

## 화면 / 라우트

| 라우트 | 페이지 | 비고 |
|---|---|---|
| `/cs/notice` | `NoticeListPage` → `Notice` | 공지사항 목록 |
| `/cs/qna` | `QnaListPage` → `QnaList` | 내 문의 목록 |
| `/cs/qna/new` | `QnaNew` | 문의 작성 (`attachment_url` 첨부 1개) |

FAQ 전용 라우트는 `App.tsx`에서 별도로 확인되지 않음 — `Notice`/`QnaList` 화면 내부에 탭으로
통합돼 있을 수 있으니 실제 컴포넌트를 확인할 것.
