# 상품/카테고리

> 확정 명세 문서가 없어 `src/types/product.ts`, `src/mocks/products.ts`,
> `src/stores/productStore.ts`, `src/pages/user/ProductList*.tsx`,
> `src/pages/user/ProductDetail*.tsx`를 기준으로 정리한 작업용 개요입니다.

## 관련 테이블 / 타입

- `category` → `Category` — `id`, `name`, `sort_order`, `deleted_at`(소프트 삭제)
- `product` → `Product`
  - `status`: `'ACTIVE' | 'INACTIVE' | 'DELETED'`
  - **옵션 없음** — 상품에 옵션 테이블이 없다. 수량만 존재.
  - **이미지 1장** — `product_image` 대표 이미지 1장을 `image_url` 필드로 단순화해서 사용.
    다중 이미지 갤러리 UI를 만들지 말 것.
  - `stock`, `price`, `category_id` 등은 스키마 그대로.

## mock 데이터 / 상태

- `src/mocks/products.ts` — 정적 상품 목록 (스낵/건강식품/초콜릿 카테고리 예시 데이터 포함).
- `src/stores/productStore.ts` — 목록 조회 상태(`products`, `selectedProduct`, `isLoading`,
  `error`, 페이지네이션 `currentPage`/`pageSize`/`total`, 필터 `categoryId`/`sort`/`keyword`).
  `sort`는 `'latest' | 'price_asc' | 'price_desc' | 'popular'` 4종.

## 화면 / 라우트

| 라우트 | 페이지 | 기능 |
|---|---|---|
| `/products` | `ProductListPage` → `ProductList` | 카테고리 필터, 정렬, 검색 |
| `/products/:productId` | `ProductDetailPage` → `ProductDetail` | 상세, 수량 선택, 장바구니/바로구매 |

품절(`stock === 0`) 및 "임박"(재고 소량) 배지가 목록/상세에 노출된다 (mock 데이터 기준).
