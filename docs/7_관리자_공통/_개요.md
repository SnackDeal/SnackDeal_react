# 관리자 공통

> 확정 명세 문서가 없어 `src/stores/adminAuthStore.ts`, `src/components/layout/AdminLayout.tsx`,
> `src/pages/admin/*`를 기준으로 정리한 작업용 개요입니다.

## 인증

- 회원(`Member`) 인증과 완전히 분리된 `AdminSession`(`src/stores/adminAuthStore.ts`)을 사용한다.
  `role`: `'ADMIN' | 'SUPER_ADMIN'` — 일반 회원의 `role: 'ADMIN'`(`Member.role`,
  [`../1_회원_인증/_개요.md`](../1_회원_인증/_개요.md))과 이름은 겹치지만 별개 개념이니 혼동하지 말 것.
- `/admin/login`에서 로그인, 이후 `AdminLayout` 하위 라우트는 `adminSession`이 없으면
  로그인으로 리다이렉트되는 구조로 추정 — 실제 가드 위치는 코드에서 확인.

## 레이아웃 / 공통 UI

- `src/components/layout/AdminLayout.tsx` — 관리자 화면 공통 레이아웃(사이드 네비게이션 등).
- `src/components/ui/`의 `Table`, `Modal`, `Select`, `Badge` 등이 관리자 목록/폼 화면에서
  공통으로 재사용된다.
