# 장바구니

> 확정 명세 문서가 없어 `src/types/cart.ts`, `src/stores/cartStore.ts`,
> `src/pages/user/Cart*.tsx`를 기준으로 정리한 작업용 개요입니다.

## 관련 테이블 / 타입

- `cart_item` → `CartItem` (`src/types/cart.ts`)
  - 옵션이 없으므로 장바구니 항목 식별 단위는 **`member_id` + `product_id`** 조합이다.
    같은 상품을 다시 담으면 새 행이 아니라 기존 행의 `quantity`를 갱신해야 한다.
  - 필드: `id`, `quantity`, `member_id`, `product_id`, `created_at`, `updated_at`.

## 상태 관리

- `src/stores/cartStore.ts`는 화면 표시에 필요한 값을 상품 스냅샷 형태로 들고 있다
  (`product_name`, `product_image`, `price`, `max_stock`, `is_soldout` 등) — 이는 `cart_item`
  테이블 자체의 필드가 아니라 상품 조인 결과를 mock에서 미리 합쳐둔 형태이므로, 실제 API 연동 시
  이 스토어의 셰이프를 그대로 응답 스키마로 착각하지 말 것.
- 로컬스토리지에 `persist`되어 새로고침해도 유지된다.

## 화면 / 라우트

| 라우트 | 페이지 | 비고 |
|---|---|---|
| `/cart` | `CartPage` → `Cart` | 로그인 필요, 수량 변경/삭제, 총액 계산 |

체크아웃(`/checkout`)으로 넘어갈 때 장바구니 선택 항목이 주문 도메인으로 전달된다 —
자세한 흐름은 [`../4_주문_결제_배송/_개요.md`](../4_주문_결제_배송/_개요.md) 참고.
