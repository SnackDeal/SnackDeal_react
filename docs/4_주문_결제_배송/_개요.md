# 주문/결제/배송/주소록

> 확정 명세 문서가 없어 `src/types/order.ts`, `src/stores/orderStore.ts`,
> `src/stores/deliveryStore.ts`, `src/pages/user/Checkout*.tsx`,
> `src/pages/user/OrderList.tsx`, `src/pages/user/OrderDetail.tsx`,
> `src/pages/user/DeliveryBook*.tsx`를 기준으로 정리한 작업용 개요입니다.

## 관련 테이블 / 타입 (`src/types/order.ts`)

- `orders` → `Order` — 주문 헤더. `status`는 8단계:
  `PENDING_PAYMENT` → `PAYMENT_COMPLETED` → `PREPARING_SHIPMENT` → `SHIPPED` → `COMPLETED`,
  그리고 `CANCELLED` / `REFUND_REQUESTED` / `REFUND_COMPLETED`.
  금액은 `product_amount`(상품가), `shipping_fee`(배송비), `discount_amount`(쿠폰 할인),
  `final_amount`(최종 결제액)로 분리.
- `order_item` → `OrderItem` — **스냅샷**. `product_name`/`price`는 주문 시점 값을 복사해서
  저장한다(상품이 나중에 바뀌어도 주문 내역은 불변). 상품 상세 정보를 다시 조회해서 채우지 말 것.
- `shipping` → `Shipping` — 주문 1건당 배송 정보 스냅샷(`order_id` UNIQUE).
  `status`: `'READY' | 'PREPARING' | 'SHIPPING' | 'DELIVERED'` (주문 상태와는 별개의 상태값).
- `payment` → `Payment` — 결제 정보(`order_id` UNIQUE). PG사는 **토스페이먼츠**
  (`pg_provider: 'tosspayments'`) — 코드/주석에 "포트원 → 토스페이먼츠"로 전환 이력이 남아있음.
  실제 결제는 mock 처리(콘솔 로그), 실 PG 연동 없음.
- `delivery` → `Delivery` — **재사용 주소록**. `shipping`(주문별 스냅샷)과는 별개 테이블이며,
  회원이 여러 배송지를 등록해두고 주문 시 선택하는 용도. `is_default`로 기본 배송지 표시.

## 화면 / 라우트

| 라우트 | 페이지 | 비고 |
|---|---|---|
| `/checkout` | `CheckoutPage` → `Checkout` | 장바구니 → 주문 생성, 배송지 선택, mock 결제 |
| `/mypage/orders` | `OrderList` | 주문 목록 |
| `/mypage/orders/:id` | `OrderDetail` | 주문 상세, 상태 표시 |
| `/mypage/delivery` | `DeliveryBookPage` → `DeliveryBook` | 배송지(주소록) CRUD |
